# 보안 이벤트 4624·4634

둘 다 **Microsoft-Windows-Security-Auditing** 공급자의 **보안** 채널 이벤트이며, **성공 감사**가 켜져 있어야 수집됩니다.

## 4624 — 계정에 로그온되었습니다

- **의미**: 새 로그온 세션이 **성공**적으로 만들어졌을 때
- **자주 쓰는 EventData 필드**:
  - **Subject\*** — 요청 주체(어떤 계정이 로그온을 시도했는지의 일부 시나리오)
  - **TargetUserName** / **TargetDomainName** — 로그온된 **대상 사용자**
  - **LogonType** — [[Logon Type 로그온 유형]]
  - **LogonProcessName** — [[LogonProcessName과 LSA 로그온 프로세스]]
  - **AuthenticationPackageName** — Negotiate, Kerberos 등
  - **WorkstationName**, **IpAddress** 등

> [!note] 한 번의 “로그온”에 4624가 여러 번
> 제한/비제한 토큰, 연결된 로그온 ID 등으로 **동일 시각에 4624가 복수** 찍힐 수 있습니다. “출근 시각”을 하나로 고를 때는 **가장 이른 시각** 또는 **추가 필터**가 필요합니다.

공식 설명: [Event 4624](https://learn.microsoft.com/en-us/windows/security/threat-protection/auditing/event-4624)

## 4634 — 계정에서 로그오프되었습니다

- **의미**: 로그온 세션이 **종료**되었을 때
- **Win+L 잠금**만 한 경우에는 **대화형 세션이 완전히 끝난 것이 아니라서** 4634가 **안 찍히는** 경우가 많습니다. **로그아웃·재부팅** 등에서 더 자주 의미가 분명합니다.

4624와 마찬가지로 **LogonType**, **TargetUserName** 등이 붙는 경우가 많아, [[GetWorkTime 필터와 이벤트 읽기 이슈]]에서는 “오늘 마지막 4634 이후의 첫 4624”로 세션 경계를 근사합니다.

공식 설명: [Event 4634](https://learn.microsoft.com/en-us/windows/security/threat-protection/auditing/event-4634)

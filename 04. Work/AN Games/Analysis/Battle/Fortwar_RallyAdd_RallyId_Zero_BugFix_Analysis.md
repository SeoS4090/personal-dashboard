# Fortwar 집결 난입 rally_id=0 버그 수정 (AWCS-1608)

## 요약

### 무슨 문제였나?
요새전에서 화면을 드래그하여 집결 전투에 난입(join)하려 할 때, 집결 ID가 0으로 잘못 전달되어 서버가 집결 처리를 거부하는 버그가 발생했다. 유저는 난입 조작을 했지만 실제로는 집결에 참가되지 않는 현상이었다.

### 어떻게 고쳤나?
| 구분 | 수정 전 | 수정 후 |
|------|---------|---------|
| rally_id 출처 | fleet 데이터의 초기값(0) 사용 | 서버로부터 명시적으로 수신한 rally_id 사용 |
| 드래그 난입 케이스 | rally_id 미설정 상태로 처리 진행 | rally_id를 서버 응답에서 읽어 명시적 설정 후 처리 |

드래그 집결 난입 처리 경로에서 rally_id를 서버 응답으로부터 직접 받아 사용하도록 수정했다.

### 현재 상태는?
- ✅ 요새전 드래그 집결 난입 시 rally_id=0 오전달 해결 완료

> **한 줄 요약**: "드래그 난입 시 초기화 전 rally_id를 읽던 경로를 서버 수신값으로 교체하여 집결 참가 실패 수정"

## 개요

- **이슈**: AWCS-1608
- **증상**: 요새전 드래그 집결 난입 시 집결이 처리되지 않음 (서버에서 rally_id=0 거부)
- **영향 범위**: 요새전(Fortwar) 드래그 조작을 통한 집결 난입 케이스

---

## 원인 분석

### 버그 발생 흐름

```
드래그로 집결 위치에 군단 이동
  └─ 집결 난입(RallyAdd) 처리 시작
       └─ rally_id 값 조회
            └─ [문제] fleet 데이터의 world_gather_slot 또는 rally_id 필드
                      드래그 난입 케이스에서 초기화 전 0값 사용
                      → 서버에 rally_id=0 전달 → 처리 실패
```

### 핵심 원인

드래그로 집결에 난입하는 흐름은 일반 집결 난입과 코드 경로가 다르다. 일반 경로에서는 fleet 데이터에 rally_id가 먼저 설정된 후 처리가 진행되지만, 드래그 난입 경로에서는 fleet의 `world_gather_slot` 또는 `rally_id` 필드가 설정되기 전에 값을 읽어버려 초기값인 0이 사용되었다.

`rally_id == 0`은 집결 없음을 의미하므로 서버는 해당 요청을 집결 처리로 인식하지 않고 거부한다.

---

## 수정 내용

### 수정 로직

드래그 집결 난입 처리 경로에 다음 변경 적용:

1. fleet 데이터에서 rally_id를 읽는 대신, 서버로부터 수신한 응답 데이터에서 rally_id를 직접 읽어 사용
2. 드래그 난입 케이스에서도 서버 응답 기준으로 rally_id를 명시적으로 설정한 뒤 처리 진행

### 검증 포인트
- 드래그 집결 난입 후 서버에 전달되는 rally_id가 0이 아닌 실제 집결 ID인지 확인
- 일반 집결 난입 경로는 영향 없는지 확인
- `rally_id == 0` 인 경우(집결 없음)에 해당하는 정상 케이스와 구분 처리 여부 확인

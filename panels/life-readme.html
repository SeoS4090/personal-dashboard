📅 최초 연동 설정 (OAuth 2.0)

OAuth 2.0으로 Google 계정에 직접 로그인하여 비공개 캘린더를 포함한 모든 캘린더를 가져옵니다. 아래 순서대로 한 번만 설정하면 됩니다.

  1. Google Cloud Console 접속 후 새 프로젝트 생성
  2. API 및 서비스 → 라이브러리에서 Google Calendar API 검색 후 사용 설정
  3. API 및 서비스 → OAuth 동의 화면 설정
    앱 유형: 외부 / 앱 이름 자유 입력 / 테스트 사용자에 본인 Google 계정 추가
  4. 자격 증명 → OAuth 클라이언트 ID 만들기 → 유형: 웹 애플리케이션
    승인된 JavaScript 원본에 아래 주소 추가:
    https://seos4090.github.io
    로컬에서 테스트할 경우 http://localhost도 함께 추가
  5. 생성된 클라이언트 ID 복사 →
    ⚙️ 설정 → Google Calendar → OAuth 2.0 Client ID에 붙여넣기 후 저장
  6. Google 계정 연결 버튼 클릭 → 팝업에서 계정 선택 후 권한 허용
  7. 연결 성공 후 ↻ 목록 불러오기 버튼 클릭 → Google에 등록된 모든 캘린더 자동 로드
  8. 불러온 목록에서 표시할 캘린더를 토글 스위치로 활성화
🗂️ 캘린더 목록 관리

계정 연결 후 ↻ 목록 불러오기를 누르면 Google 계정에 등록된 캘린더가 전부 나타납니다.

  • 토글 ON — 일정 패널에 해당 캘린더의 일정이 표시됩니다
  • 토글 OFF — 목록에는 유지되지만 일정 패널에서 숨겨집니다
  • 캘린더별 색상은 Google Calendar에 설정된 색상을 그대로 사용합니다
  • 새 캘린더를 Google에서 추가했다면 ↻ 목록 불러오기로 갱신하세요
  • 인증이 만료된 경우 Google 계정 연결을 다시 클릭해 재인증합니다

💡 대한민국 공휴일을 표시하려면 Google Calendar 앱에서 다른 캘린더 → 관심 있는 캘린더 찾아보기 → 대한민국을 구독한 후 목록을 다시 불러오세요.

🔄 연결 유지 (자동 토큰 복원)

한 번 연결하면 새로고침(F5)이나 탭 재방문 시에도 자동으로 연결 상태가 복원됩니다.

  • 연결 후 액세스 토큰이 브라우저에 저장되며, 58분간 유효합니다
  • 유효한 토큰이 남아 있으면 새로고침해도 팝업 없이 캘린더가 바로 표시됩니다
  • 토큰이 만료되어도 Google 로그인 세션이 살아 있으면 팝업 없이 자동 재인증합니다
  • Google 세션까지 만료된 경우에만 "다시 연결" 버튼이 표시됩니다
  • Client ID를 변경하면 저장된 토큰이 초기화되고 재인증이 필요합니다

💡 토큰은 이 브라우저의 localStorage에만 저장됩니다. 다른 브라우저나 기기에서는 별도로 연결해야 합니다.

🗓️ 뷰 전환 및 탐색
  • 월별 — 달력 그리드로 한 달 일정을 한눈에 확인. 날짜 칸에 일정 점(dot)으로 표시
  • 주별 — 7일 타임라인 뷰 (오전 7시 ~ 오후 10시). 시간대 겹침 일정도 표시
  • 일정 목록 — 오늘부터 30일간 일정을 날짜별로 그룹핑하여 목록 표시
  • ‹ › 버튼으로 이전 / 다음 기간으로 이동
  • 상단 날짜 표시 클릭 시 오늘로 빠르게 복귀
  • 주 시작 요일은 ⚙️ 설정 → 주 시작 요일에서 일요일 / 월요일 선택 가능
⚠️ 문제 해결
  • 팝업이 차단됨 — 브라우저 주소창 우측의 팝업 차단 아이콘을 클릭해 허용 후 재시도
  • "이 앱은 Google에서 확인하지 않았습니다" — 고급 → (앱 이름)(으)로 이동 클릭. OAuth 동의 화면이 테스트 상태일 때 표시되는 정상 메시지
  • 인증 만료 오류 — Google 세션이 살아 있으면 자동으로 재인증됩니다. "다시 연결" 버튼이 표시되는 경우에만 클릭해 재인증하세요
  • 캘린더 일정이 안 보임 — 설정에서 해당 캘린더 토글이 ON인지 확인. Client ID 저장 후 저장 버튼을 눌렀는지도 확인
  • 승인된 원본 오류 (origin mismatch) — Google Cloud Console 자격 증명에서 현재 접속 주소가 승인된 JavaScript 원본에 등록되어 있는지 확인
📰 NewsAPI.org 연동

NewsAPI.org를 통해 키워드 기반 뉴스를 실시간으로 가져옵니다.

  1. newsapi.org 회원가입 (무료 플랜 제공)
  2. 로그인 후 My Account 페이지에서 API Key 복사
  3. 대시보드 ⚙️ 설정 → News API에 API Key 입력
  4. 관심 키워드를 쉼표로 구분해 입력 (예: 게임, AI, 개발, 스타트업)
  5. 저장 후 뉴스 패널로 이동하면 자동으로 로드됩니다
⚠️ 무료 플랜 제한사항
  • 무료 플랜은 하루 100회 API 호출 제한 (일반 사용에 충분)
  • 기사 본문은 일부만 제공 — 전문은 카드 클릭 후 원본 링크에서 확인
  • 해외 뉴스 중심이므로 영문 키워드를 함께 추가하면 더 많은 결과를 얻을 수 있습니다
  • 한국 뉴스 위주로 받으려면 키워드에 korea를 포함하거나 한글 키워드 사용
📋 Srello 사용법

Trello를 차용한 개인 칸반 보드입니다. 리스트(열)와 카드로 작업을 관리합니다.

  • + 카드 추가 — 각 리스트 하단에서 카드 제목을 빠르게 추가
  • 카드 클릭 — 제목·설명·라벨 색상 편집 및 삭제
  • 드래그 앤 드롭 — 카드를 다른 리스트로 옮기거나 같은 리스트 안에서 순서 변경
  • 리스트 제목 — 상단 입력란을 클릭해 이름 변경 (Enter 또는 포커스 아웃 시 저장)
  • + 리스트 추가 — 보드 맨 오른쪽에서 열 추가
  • 기본 리스트: 할 일 · 진행 중 · 완료
🃏 Trello 기능 vs Srello

보드는 카테고리(리스트), 카드 색·P0 뱃지는 우선순위에 대응합니다.

Trello Srello
Board / List / Card보드 / 리스트(카테고리) / 카드
LabelsP0–P3 색 + 뱃지
Due dates · Checklists🚧 예정 (P0)
Comments · Attachments🚧 백로그
Butler · Calendar · Templates📋 Srello 리스트 백로그

우선순위: P0 긴급 · P1 높음 · P2 보통 · P3 백로그

🤖 Claude 모니터

여러 Claude Code 계정의 작업량과 토큰 리셋 시각을 한눈에 보고, 각 계정의 상태를 귀여운 CSS 캐릭터(일하는 / 쉬는 / 곧 리셋)로 시각화합니다.

  • ＋ 계정 추가 — 이름 · 플랜 · 아바타 · 색상 · 리셋 주기(기본 5시간) 설정
  • ▶ 세션 시작 — 누르면 사용량 창이 시작되고 리셋까지 카운트다운이 돌아갑니다 (캐릭터가 일하기 시작)
  • 리셋 시각이 지나면 “사용 가능 ✓”로 바뀌고 캐릭터는 쉬는 상태(💤)가 됩니다
  • 오늘 메시지 · 세션 · 토큰과 최근 7일 막대그래프 표시
🔌 실데이터 연동 (로컬 전용)

브라우저(특히 GitHub Pages 배포본)는 보안상 로컬 파일을 읽거나 claude.ai/Anthropic 사용량을 직접 조회할 수 없습니다. 실제 수치는 로컬에서 sync 스크립트를 돌릴 때만 채워집니다.

  • node scripts/claude-usage-sync.js 실행 → data/claude-usage.generated.json 생성
  • 각 계정의 ~/.claude/stats-cache.json(메시지·세션·토큰)을 읽어 집계합니다
  • 여러 계정은 scripts/claude-accounts.config.json에 계정별 configDir(CLAUDE_CONFIG_DIR 경로)를 등록 — 예시는 *.config.example.json
  • 계정 카드의 실데이터 연동 키를 generated/example의 계정 key와 맞추면 수치가 연결됩니다
  • 개인 사용량(generated.json)·로컬 경로(config.json)는 .gitignore — 커밋되지 않습니다
  • 리셋 시각은 (Windows) 기존 ClaudeCodeReset 예약 작업이 있으면 자동 감지, 없으면 수동 카운트다운

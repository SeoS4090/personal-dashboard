# TODO-리스트-가이드

기획 작업의 **실행 목록**은 `TODO list/` 에 모은다. 원자 노트(`notes/`)는 규칙·설계 본문만 두고, «지금 무엇을 해야 하는가»는 [[할일-인덱스]] 만 보면 된다.

## 폴더 역할

| 경로 | 용도 |
|------|------|
| `TODO list/할일-인덱스.md` | **유일한 실행 진입점** — 전체·우선순위·진행률 |
| `TODO list/*-할일.md` | 도메인별 체크리스트 (MOC와 1:1 대응) |
| `notes/*/…` | 원문 노트 하단 `## 5. 미결·TODO` — 맥락·세부 조건 |

## 노트를 만들거나 수정할 때 (필수)

1. 원자 노트에 할 일이 생기면 `## 5. 미결·TODO` 에 `- [ ]` 항목 추가
2. **같은 항목**을 해당 도메인 `TODO list/*-할일.md` 에도 추가하고, 원문으로 링크  
   예: `- [ ] 보스 면역 테이블 → [[상태이상-기절#5. 미결·TODO|기절 노트]]`
3. [[할일-인덱스]] 요약 표·진행률 갱신 (완료 시 `[x]` + 인덱스 동기화)
4. 원문 노트 `updated` 갱신

## 완료 처리

- 체크는 **할일 문서와 원문 노트 둘 다** `[x]` (한쪽만 하지 않음)
- 설계가 확정되면 원문 `status: confirmed` 검토, 할일 항목은 인덱스에서 제거하거나 «완료» 섹션으로 이동

## AI·에이전트

- «다음에 뭘 하지?» → [[할일-인덱스]] → 해당 `*-할일.md` 만 Read
- 기획 규칙·수치 확인 → [[Index]] → 도메인 MOC → 원자 노트
- 저장소 `AGENTS.md` 체크리스트에 TODO 동기화 포함

## 새 도메인 MOC 추가 시

1. `TODO list/<도메인>-할일.md` 생성 (`_templates/todo-list.md`)
2. [[할일-인덱스]] 표에 행 추가
3. 도메인 MOC 개요에 «할 일: [[<도메인>-할일]]» 한 줄 링크

# 기획 문서 작성 방안

Obsidian으로 기획 라이브러리를 구축하고, AI가 잘 이해하도록 구조화하는 접근 요약이다. 실행 규격은 [[프론트매터-규격]], [[Atomic-Notes-가이드]], [[AI-워크플로우]] 를 따른다.

## AI 친화적인 설계 룰 5가지

### 1. 원자적 노트 (Atomic Notes)

**1노트 1개념.** 거대한 `전투 시스템.md` 대신 `전투 스킬 시스템`, `대미지 계산 공식`, `상태이상-기절` 등으로 분리하고 링크한다.

- 나쁜 예: 한 파일에 기본 공격·스킬·상태이상·UI·공식 전부
- 좋은 예: 개념별 노트 + [[전투 MOC]] 로 묶기

AI는 "기절일 때 대미지 공식?" 같은 질문에 관련 노트만 골라 답할 수 있다 (RAG 효율).

### 2. YAML 프론트매터 규격화

모든 노트 상단에 정체성 메타데이터. 상세 필드는 [[프론트매터-규격]] 참고.

```yaml
---
type: system
status: draft
domain: "[[전투 MOC]]"
related:
  - "[[대미지 계산 공식]]"
updated: 2026-05-20
---
```

예: "초안 상태 전투 system 문서만 모순 검토" 같은 고차원 필터가 가능하다.

### 3. 일관된 헤딩 템플릿

문서 `type` 별로 `##` 구조를 통일한다. 시스템 노트는 `_templates/system.md` 구조를 따른다.

1. 기획 의도 및 핵심 루프  
2. 핵심 규칙 (Core Rules)  
3. 예외 처리 및 엣지 케이스  
4. 데이터 구조 및 인터페이스  

### 4. 문장 속 양방향 링크

관계는 목록이 아니라 **문장 안** `[[위키링크]]` 로 정의한다.

> 캐릭터가 사망하면 [[재화 시스템]]의 [[골드]]가 10% 차감되며, 부활 시 [[소모성 아이템 시스템]]의 [[부활 주문서]]를 사용할 수 있다.

### 5. MOC 중심 인덱싱

폴더는 1~2단계만. 탐색은 [[Index]] → 도메인 MOC → 원자 노트.

### 6. TODO list (실행 목록)

«무엇을 할 것인가»는 [[할일-인덱스]] (`TODO list/`) 에만 모은다. 원자 노트 `## 5. 미결·TODO` 와 **항목 단위로 동기화** ([[TODO-리스트-가이드]]).

MOC는 링크 나열이 아니라 요약 문단 + 표·링크:

> 우리 게임의 경제는 [[재화 시스템]]을 중심으로 [[상점 시스템]]·[[보상 시스템]]과 연동된다.

## Obsidian 세팅 (권장)

1. **Linter** — 줄바꿈·헤딩·프론트매터 정렬  
2. **Templater** — `기획/_templates/` 유형별 생성  

## 이 볼트에서의 다음 단계

- [[Index]] 에 도메인 MOC 추가 (경제, UI, 세계관 등)
- 예시 노트 [[대미지 계산 공식]], [[상태이상-기절]] 를 실제 기획으로 교체
- `프로그램/` 에 엔진 프로젝트 생성

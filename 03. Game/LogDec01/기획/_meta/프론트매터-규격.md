# 프론트매터 규격

모든 **기획 노트**(`notes/`, `MOC/`) 상단 YAML은 아래 필드를 사용한다. AI가 문서 정체성을 파악하는 1차 신호다.

## 필수·권장 필드

| 필드 | 필수 | 타입 | 설명 |
|------|------|------|------|
| `type` | ✅ | enum | 문서 유형 (아래 표) |
| `status` | ✅ | enum | `draft` \| `review` \| `confirmed` |
| `title` | 권장 | string | 표시 제목 (파일명과 다를 수 있음) |
| `domain` | ✅ | string / link | 소속 MOC. 예: `[[전투 MOC]]` 또는 `combat` |
| `updated` | ✅ | date | `YYYY-MM-DD` 최종 수정일 |
| `tags` | 권장 | list | Obsidian 태그 |
| `related` | 권장 | list | 관련 노트 위키링크 문자열 |
| `aliases` | 선택 | list | 링크 자동완성용 별칭 |

## `type` 값

| 값 | 용도 | 저장 위치 |
|----|------|-----------|
| `moc` | Map of Content | `MOC/`, `00_Index/` |
| `system` | 시스템 기획 (규칙·루프·연동) | `notes/` |
| `content` | 컨텐츠 (퀘스트·캐릭터·연출) | `notes/` |
| `data` | 데이터 테이블·스키마 | `notes/` |
| `worldview` | 세계관·설정 | `notes/` |
| `todo` | 실행 체크리스트 (할 일만) | `TODO list/` |
| `meta` | 규격·가이드 (`_meta/` 만) | `_meta/` |

## 예시 — 시스템 노트

```yaml
---
type: system
status: draft
title: 대미지 계산 공식
domain: "[[전투 MOC]]"
tags:
  - combat
  - formula
related:
  - "[[상태이상-기절]]"
  - "[[캐릭터 스탯 규격]]"
aliases:
  - 데미지 공식
updated: 2026-05-20
---
```

## 예시 — MOC

```yaml
---
type: moc
status: confirmed
title: 전투 MOC
domain: combat
tags:
  - moc
  - combat
updated: 2026-05-20
---
```

## AI 필터 예시

- "확정된 전투 시스템만": `type: system` + `domain` 전투 + `status: confirmed`
- "초안만 검수": `status: draft`
- "이 노트와 직접 연결": `related` 및 본문 `[[링크]]` 그래프

## 금지·주의

- 프론트매터에만 링크하고 본문에 관계를 쓰지 않는 것 — **본문 문장 링크를 우선**하고 `related`는 보조 인덱스로 둔다.
- `type` 없이 장문만 있는 파일 — 정체성 불명, AI 검색 효율 저하.
- 한 노트에 `type: system` 과 `type: content` 를 혼합하는 것 — **노트를 분리**한다.

## Obsidian 플러그인 (권장)

- **Linter**: 줄바꿈·헤딩·프론트매터 정렬
- **Templater**: `_templates/` 로 유형별 생성 자동화

# Atomic Notes 가이드

**1노트 = 1개념.** AI(RAG·에이전트)는 거대 단일 문서보다 맥락이 분리된 작은 노트를 더 정확히 검색·추론한다.

## 나쁜 예 vs 좋은 예

| 나쁜 예 | 좋은 예 |
|---------|---------|
| `전투 시스템.md` 에 공격·스킬·UI·공식 전부 | `전투 스킬 시스템`, `대미지 계산 공식`, `상태이상-기절` 각각 분리 후 링크 |
| 절 제목만 있는 목차형 장문 | 각 절이 독립 노트가 될 만큼 크면 분리 |

## 노트 크기 가이드

- **한 화면~두 화면** 분량을 넘기면 분할 후보
- 제목이 "및", "와", 쉼표로 여러 개념을 나열하면 분할 후보
- "예외 처리"만 길어지면 → `[[시스템명-예외 케이스]]` 로 분리 가능

## 연결 방법

1. 본문: "기절 중에는 [[대미지 계산 공식]]의 크리티컬 배율이 0이다."
2. 프론트매터: `related: ["[[대미지 계산 공식]]"]`
3. MOC: 도메인 MOC에 한 줄 요약 + `[[노트]]`

## 파일명·경로

- 파일명 = 개념명 (공백 허용, Obsidian 기본)
- `notes/` 아래는 **유형별 최대 1단계**만 사용:
  - `notes/system/`, `notes/content/`, `notes/data/`, `notes/worldview/`
- 더 깊은 폴더 대신 **MOC + tags + domain** 으로 분류

## 새 노트 체크리스트

- [ ] 하나의 질문에 답할 수 있는가? ("X는 언제 Y인가?")
- [ ] `type`·`domain`·`status` 를 넣었는가?
- [ ] 관련 MOC에 링크를 추가했는가?
- [ ] 문장 속 `[[위키링크]]` 로 관계를 적었는가?

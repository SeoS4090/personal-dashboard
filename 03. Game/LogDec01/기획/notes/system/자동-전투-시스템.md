# 자동-전투-시스템

## 1. 기획 의도 및 핵심 루프

- 전투 중 플레이어 입력 없이 [[덱-구성-규칙]] 의 슬롯·덱이 스스로 순환 발동한다.
- 플레이어는 전투 **전** 슬롯 순서·덱 구성·[[상점-시스템]] 으로 «AI가 얼마나 자주 무엇을 쓸지»를 설계한다.

## 2. 핵심 규칙 (Core Rules)

### 슬롯 순환 (손패 없음)

| 항목 | 값 |
|------|-----|
| 슬롯 수 | 8 (유물로 +1, 최대 12) |
| 틱 간격 | 0.5초 (표시용; 로직은 라운드 단위 가능) |
| 1 라운드 | 글로벌 틱 10회 |

**발동 순서**

1. 전투 시작: 덱에서 카드를 슬롯 1~8에 배치(플레이어가 사전에 정한 슬롯 순서·덱 목록)
2. 매 틱: 쿨다운 0인 카드 중 **슬롯 번호 오름차순** 1장 발동
3. 발동 후: 해당 카드 쿨다운 = `cost`(1~4) 라운드
4. 동일 틱에 적도 [[대미지 계산 공식]] · speed 규칙으로 행동

### speed·행동 횟수

- 유닛 `speed` 기본 10
- 라운드당 행동 횟수 = `max(1, floor(speed / 10))`
- [[상태이상-둔화]] 시 speed -30% (중첩 2)

### 타겟 AI

카드 `target` + 직업 패시브 가중.

| target | 동작 |
|--------|------|
| `nearest` | 전방 단일 |
| `lowest_hp` | HP% 최저 |
| `random` | 무작위 1체 |
| `all_enemies` | 전체 |
| `self` | 자신 |

- [[직업-궁수]]: `lowest_hp` 가중 +20%
- [[직업-전사]]: `nearest` 고정(도발 카드는 적의 타겟을 자신으로)

### 기절·행동 스킵

- [[상태이상-기절]] 중에는 카드 발동·일반 공격 스킵

### 전투 UI (도트)

- 좌: 플레이어 / 우: 적, HP·보호막 바, 상태 아이콘
- 하단: 8슬롯 + 쿨다운 숫자, 스크롤 로그
- 속도: ×1 / ×2 / ×4

## 3. 예외 처리 및 엣지 케이스

- 슬롯에 카드 미장착: 해당 슬롯 스킵
- 전투 중 덱 변경 없음(상점·이벤트만)
- 동시 사망: 플레이어 패배 우선

## 4. 데이터 구조 및 인터페이스

- 전투 상태: round, tick, slots[{cardId, cooldown}], units[]
- 카드 필드: [[카드-스키마]]

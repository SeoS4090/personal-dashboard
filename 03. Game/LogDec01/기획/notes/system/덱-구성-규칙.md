# 덱-구성-규칙

## 1. 기획 의도 및 핵심 루프

- 로그라이크 덱 빌딩의 성장 축. [[자동-전투-시스템]] 은 덱을 **8~12 슬롯**에 올려 순환 발동한다.
- [[상점-시스템]] 과 전투 보상으로 카드 추가·제거·업그레이드한다.

## 2. 핵심 규칙 (Core Rules)

| 규칙 | 값 |
|------|-----|
| 시작 덱 | 직업별 10장 ([[직업-시작-덱-표]]) — 공격 6 + 방어/유틸 4 |
| 덱 상한 | 없음 |
| 덱 하한 | 전투 시작 시 최소 5장; 미달 시 `strike_basic` 자동 보충 |
| 전투 중 손패 | 없음 |
| 슬롯 | 8 기본, 유물로 +1 (최대 12) |
| 슬롯 순서 | 플레이어가 맵/상점 화면에서 드래그 정렬 → 발동 우선순위 |

### 카드 획득·제거

- 상점 카드 칸 2개, 전투 보상(엘리트), 이벤트
- 제거: 상점 덱 서비스 75G ([[재화-골드]])
- 업그레이드: 100G, 동일 카드 id의 `+` 버전으로 교체

### 희귀도

- 일반 / 고급 / 희귀 — 상점·보상 가중치 상이 ([[보상-시스템]])

## 3. 예외 처리 및 엣지 케이스

- 동일 카드 3장 이상 중복 허용
- 슬롯 > 덱 장수: 빈 슬롯은 스킵

## 4. 데이터 구조 및 인터페이스

- 런 덱: `cardId[]`
- 슬롯: `slotOrder: cardId[]` (길이 ≤ 슬롯 수)
- [[카드-스키마]] 참조

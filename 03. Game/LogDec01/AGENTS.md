# Agent 작업 규칙 (LogDec01)

이 프로젝트에서 AI·에이전트가 기획 문서를 읽거나 수정할 때 따를 규칙입니다.

## 1. 폴더 역할

| 경로 | 용도 | 수정 가능 |
|------|------|-----------|
| `기획/` | Obsidian 기획 볼트 | 기획 문서만 |
| `프로그램/` | 개인 홈페이지 (HTML/CSS/JS) | 웹·에셋만 |
| `기획/_meta/` | 규격·가이드 | 구조 변경 시에만 |
| `기획/TODO list/` | 실행 체크리스트 (할 일만) | 할 일 문서만 |

**기획 문서를 프로그램 폴더에 두지 않는다.** **기획 장문을 프로그램에 두지 않는다.**

홈페이지 구조: `프로그램/index.html` → `games/` → 개별 프로젝트. 새 게임은 `site-config.js` + `games/<id>/index.html`.

## 2. MOC 우선 탐색 (필수)

문서를 읽거나 쓸 때 **폴더 트리를 훑지 말고 MOC를 따른다.**

1. `기획/00_Index/Index.md` 에서 작업 도메인 MOC 확인
2. 해당 `기획/MOC/*.md` 에서 관련 원자 노트 링크만 추린다
3. **필요한 노트만** `기획/notes/` 에서 Read/Edit
4. 변경 후 **연결된 MOC·관련 노트의 링크·메타**가 맞는지 확인

새 개념 추가 시: 노트 생성 → 관련 MOC에 링크·한 줄 요약 추가 → `related` / 본문 위키링크로 양방향 연결.

**«다음에 뭘 하지?»** → `기획/TODO list/할일-인덱스.md` 만 연다. 규칙 확인이 필요할 때만 MOC → 원자 노트.

## 2-1. TODO list (실행 목록)

1. 실행 진입: `기획/TODO list/할일-인덱스.md`
2. 도메인별: `기획/TODO list/<도메인>-할일.md`
3. 원자 노트 `## 5. 미결·TODO` 에 항목을 넣으면 **같은 항목**을 해당 `*-할일.md` 에도 넣고 원문으로 `[[링크]]`
4. 완료 시 할 일 문서·원문 노트 **둘 다** `[x]`, [[할일-인덱스]] 진행 표 갱신
5. 규격: `기획/_meta/TODO-리스트-가이드.md` · 템플릿 `_templates/todo-list.md`

## 3. Atomic Notes

- 한 파일에 여러 시스템·여러 규칙을 넣지 않는다.
- 제목 = 하나의 개념 (예: `대미지 계산 공식`, `상태이상-기절`).
- 거대 문서가 필요해 보이면 **MOC로 쪼개고** 노트를 분리한다.

## 4. 프론트매터

모든 `기획/notes/`·`기획/MOC/` 마크다운은 `기획/_meta/프론트매터-규격.md` 를 따른다.

- `type`, `status`, `domain`, `updated` 는 필수에 가깝게 유지
- AI 필터 예: `status: draft` 인 `type: system` 만 검토

## 5. 링크·임베드

- 관계는 **문장 안** `[[노트명]]` 으로 쓴다 (단순 목록 링크만 금지하지 않음, 문장 링크 권장).
- MOC는 요약 문단 + 링크 목록을 함께 둔다.
- 노트 간 `related` 프론트매터와 본문 링크를 일치시킨다.

## 6. 수정 체크리스트

- [ ] MOC에서 진입했는가
- [ ] 1노트 1개념인가
- [ ] 프론트매터·`updated` 갱신했는가
- [ ] 관련 MOC·노트에 링크 반영했는가
- [ ] 코드 변경은 `프로그램/` 에만 있는가
- [ ] 미결·TODO를 `TODO list/` 에 동기화했는가 ([[할일-인덱스]])

## 7. 템플릿

새 파일은 `기획/_templates/` 의 해당 유형 템플릿으로 생성한다.

- MOC → `_templates/MOC.md`
- 시스템 → `_templates/system.md`
- 컨텐츠 → `_templates/content.md`
- 데이터 → `_templates/data.md`
- 할 일 목록 → `_templates/todo-list.md` → `TODO list/`

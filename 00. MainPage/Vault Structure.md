## 1. 상위 폴더 구조 개요

이 Vault(`Obsidian_Memo`)의 최상위는 크게 다음 요소들로 구성되어 있다.

- `00. MainPage` : 대시보드 및 진입점
- `02. Financial` : 가계부·급여 등 재정 기록
- `04. Work` : 업무 관련 노트 및 AI 연동 문서
- `99. Temp` : 템플릿과 임시 메모
- `.obsidian` : Obsidian 설정 및 플러그인 데이터 (앱이 사용하는 시스템 폴더)
- `.git`, `.gitignore` : 버전 관리 설정

> **원칙**: 숫자+이름 패턴으로 폴더를 정렬(00, 02, 04, 99)하여, 좌측 탐색기에서 항상 일정한 순서로 보이도록 구성되어 있다.

## 2. 각 폴더 역할 상세

### 2.1 `00. MainPage`

- 역할: Vault 전체를 조망하는 **메인 대시보드 영역**.
- 주요 파일:
  - `Dash Board.canvas`
    - 캔버스 노트로, 다른 문서/템플릿에 대한 입구 역할을 한다.
    - 예: `99. Temp/Template Daily.md`, `99. Temp/Template Schedule.md`를 그룹(`템플릿`)으로 묶어 시각적으로 배치.
- 사용 방식:
  - Obsidian을 열면 가장 먼저 참고하는 **허브(시작 화면)**로 활용.
  - 자주 쓰는 템플릿·프로젝트·리스트를 이 폴더/캔버스에서 링크로 관리.

### 2.2 `02. Financial`

- 역할: **재무·가계부·급여** 등 돈 관련 정보를 모아두는 영역.
- 예시 파일:
  - `2026.02.md`
    - 월 단위 가계부 양식.
    - 상단에 `{{date:YYYY-mm}} 가계부` 제목과, 날짜/내용/분류/수입/지출 표가 있다.
    - `### 요약` 섹션에서 총 수입·총 지출·잔액을 정리하도록 설계.
    - 하단 `### Notes`에 추가 메모를 남기는 구조.
- 사용 방식:
  - 매달 `YYYY.MM.md` 형태로 파일을 복제해 사용.
  - 표를 기반으로 수입·지출을 기록하고, 요약 섹션에 월말 정산을 정리.

### 2.3 `04. Work`

- 역할: **업무(Work)** 관련 노트를 모으는 영역.
- 현재 확인된 하위 구조:
  - `04. Work/AI/obsidian-cli-context7.md`
    - 제목: `Cursor(context7) + Obsidian CLI 활용 가이드`
    - 내용:
      - Cursor + context7를 이용해 Obsidian CLI 공식 문서(`https://help.obsidian.md/cli`)를 참고하는 방법.
      - Vault 경로(`C:\Users\admin\Desktop\Personal\Obsidian_Memo`) 기준 CLI 예시 명령.
      - Task 자동화 아이디어, 태그/링크 정리 워크플로우 등.
- 사용 방식:
  - 업무 관련 프로젝트, 도구, AI 연동 문서를 이 영역에 정리.
  - AI 관련 설정/가이드는 `04. Work/AI` 아래에 모아두는 패턴.

### 2.4 `99. Temp`

- 역할: **템플릿과 임시 노트**를 모아두는 영역.
- 주요 파일:
  - `Template Daily.md`
    - Todo 리스트 기반 일일 템플릿.
    - 예: `운동 하기`, `약 챙겨 먹기` 등 기본 체크리스트 포함.
  - `Template Schedule.md`
    - 일정/스케줄 기록에 사용할 수 있는 템플릿.
  - `How To Use.md`
    - 마크다운 사용 예시(표, 글꼴, 링크, 이미지 삽입 등)를 HTML 형태로 설명.
    - 마크다운 문법·스타일을 테스트하거나 참고할 수 있는 문서.
- 사용 방식:
  - 실제 작업용 노트를 만들 때, 이 폴더의 템플릿들을 복사해서 사용.
  - 실험/연습용 문서도 이곳에 두어 상위 구조를 깔끔하게 유지.

## 3. 시스템 폴더

### 3.1 `.obsidian`

- Obsidian 프로그램이 사용하는 설정·플러그인 데이터 저장소.
- 예시 파일:
  - `workspace.json` : 현재 열려 있는 탭, 분할 상태 등 UI 레이아웃.
  - `community-plugins.json` : 설치된 커뮤니티 플러그인 목록.
  - `appearance.json` : 테마, 글꼴 등 외형 설정.
  - `daily-notes.json` : 데일리 노트 관련 설정(경로, 파일명 패턴 등).
  - 각종 플러그인 폴더:
    - `plugins/templater-obsidian`
    - `plugins/obsidian-git`
    - `plugins/calendar`
    - `plugins/obsidian-tasks-plugin`
- 일반 노트 편집 시에는 이 폴더를 직접 수정할 필요는 없고, **Obsidian 설정 화면**을 통해 간접적으로 관리하는 것이 안전하다.

### 3.2 `.git` 및 `.gitignore`

- 이 Vault는 Git으로 버전 관리 중이다.
- `.git` 폴더:
  - 커밋 기록, 브랜치, 리모트 정보 등을 포함.
- `.gitignore`:
  - Git으로 추적하지 않을 파일/폴더(예: 캐시, 일부 플러그인 데이터 등)를 지정.
- 사용 방식:
  - Obsidian 내 `obsidian-git` 플러그인을 통해 커밋/푸시를 관리하거나,
  - 외부 Git 클라이언트(Sourcetree 등)를 통해 버전 관리를 수행.

## 4. 추천 사용 흐름

1. **메인 진입**: `00. MainPage`의 `Dash Board.canvas`에서 오늘 사용할 템플릿·프로젝트 링크로 이동.
2. **일상 기록**:
   - 일일 기록이 필요하면 `99. Temp/Template Daily.md`를 복사해 오늘 날짜 노트 생성.
   - 재무 기록은 `02. Financial` 아래 월별 파일을 사용.
3. **업무/도구 관리**:
   - 업무 관련 내용은 `04. Work` 아래에 정리.
   - AI/도구 설정·가이드는 `04. Work/AI` 아래에 두어 기술 문서와 함께 관리.
4. **구조 변경/확장 시**:
   - 새로운 상위 영역이 필요하면 `06. Something` 같은 형식으로 폴더를 추가.
   - 이 문서(`00. MainPage/Vault Structure.md`)에 변경 내용을 간단히 메모해 두면, 미래의 나/다른 사람이 구조를 이해하기 쉽다.

## 5. 이 문서를 업데이트할 때

- 폴더가 추가되거나 역할이 바뀌면:
  - 해당 폴더 설명을 2번 섹션에 추가/수정.
- 새로운 시스템/플러그인을 도입하면:
  - `.obsidian` 관련 중요 설정 파일만 간단히 소개(예: 새 플러그인 이름과 용도).

이 문서는 **Vault 전체의 지도(Overview)** 역할을 한다.  
구조를 바꾸기 전·후에 항상 이 문서를 한 번 열어보고, 변경 이력을 간단히 남겨 두면 장기적으로 Vault 유지 관리에 큰 도움이 된다.
